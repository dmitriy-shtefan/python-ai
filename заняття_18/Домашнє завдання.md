# Домашнє завдання 18. Доробка і деплой OpenRouter чатбота

**Дата заняття:** 10.07.2026 (пт)  
**Модуль:** Модуль 8. Використання наявних рішень

## Мета

Доробити AI чатбота зі Streamlit та OpenRouter REST API, підготувати repository і задеплоїти застосунок на Streamlit Community Cloud.

## Завдання

Візьміть чатбота, створеного на занятті, і зробіть 2 невеликі доробки.

### Доробка 1. Режими асистента

Додайте в sidebar вибір режиму асистента через `st.selectbox`.

Мінімум 3 режими:

1. `Python tutor` - пояснює Python простими словами і дає короткі приклади.
2. `Code reviewer` - перевіряє код і дає 2-4 конкретні поради.
3. `Study assistant` - допомагає сформулювати план навчання або повторення теми.

Для кожного режиму має змінюватися system prompt, який додається першим повідомленням у список `messages`.

### Доробка 2. Експорт історії діалогу

Додайте кнопку для завантаження історії чату у Markdown або `.txt` файл через `st.download_button`.

Файл має містити:

- повідомлення користувача;
- відповіді асистента;
- назву вибраного режиму асистента;
- назву OpenRouter-моделі, яка використовувалась.

## Деплой

Задеплойте застосунок на Streamlit Community Cloud.

Вимоги:

1. API key не має бути в коді, notebook, README або commit history.
2. Для локального запуску можна використовувати environment variable або `.env`.
3. Для Streamlit Cloud використайте Secrets і назву `OPENROUTER_API_KEY`.
4. У repository мають бути:
   - `app.py`;
   - `requirements.txt`;
   - `README.md`;
   - `.gitignore`.
5. У `requirements.txt` мають бути всі потрібні бібліотеки, зокрема `streamlit` і `requests`.
6. Якщо використовуєте `.env`, додайте `python-dotenv` у `requirements.txt`.
7. Якщо використовуєте локальний `.streamlit/secrets.toml`, додайте його в `.gitignore`.
