# Командне завдання 1. Спроєктуй MCP Server: Travel Planner

**Предметна область:** планування подорожей.

## Мета

Навчитися відрізняти MCP `resources`, `tools` і `prompts`.

## Завдання

Спроєктуйте в одному Markdown-файлі сервер **Travel Planner MCP** для помічника з планування подорожей.

Кожен учасник команди готує:

- 1 **resource** — дані для читання, наприклад опис міста, правила багажу або перелік пам’яток;
- 1 **tool** — дію, наприклад розрахунок бюджету чи підбір маршруту;
- 1 **prompt** — повторювану інструкцію для AI-помічника, наприклад «Склади маршрут на 3 дні».

У prompt використайте щонайменше один resource або tool, підготовлений іншим учасником вашої команди.

> **Роль капітана команди:** капітан не готує окремий resource, tool чи prompt. Він демонструє екран під час презентації та фіксує спільний результат у Markdown-файлі.

## Як працювати

1. Оберіть один простий сценарій: хто і куди їде, на скільки днів та з яким бюджетом.
2. Розподіліть між учасниками resources, tools і prompts так, щоб не було повторів.
3. Коротко опишіть кожен елемент: назва та одне речення, що він дає або робить.
4. Зберіть усе в один приклад: запит користувача → resource/tool → prompt → відповідь помічника (приклад можливої відповіді).

## Що показати наприкінці

- назву вашої теми та MCP сервера;
- список усіх resources, tools і prompts із зазначенням автора;
- один приклад роботи помічника;
- коротке пояснення: чому ваші приклади є саме resource, tool і prompt.
