## Мета

Закріпити створення MCP tools у FastMCP, розширивши сервер аналізу продажів із практичного заняття.

## Завдання

Візьміть за основу практичний проєкт `mcp_sales_demo` і додайте до файлу `sales_mcp_server.py` **один новий MCP tool**.

### Tool `sales_by_region`

Створіть tool `sales_by_region(file_name: str = DEFAULT_CSV) -> dict[str, object]`, який аналізує CSV-файл продажів за стовпцем `region`.

Tool має:

1. Використовувати наявну функцію `safe_csv_path` для безпечного доступу до CSV-файлу.
2. Перевіряти наявність стовпців `region`, `quantity`, `unit_price` і `cost_per_unit`.
3. Для кожного регіону порахувати:
   - кількість замовлень: кількість рядків CSV, що належать до регіону;
   - виручку в гривнях: `сума(quantity × unit_price)` для всіх замовлень регіону;
   - валовий прибуток у гривнях: `сума(quantity × (unit_price - cost_per_unit))` для всіх замовлень регіону.
4. Повернути результат як словник із назвою файлу та списком регіонів, відсортованим за виручкою від більшої до меншої.

Перевірте, що новий tool з'являється серед capabilities MCP-сервера і повертає результат для `sales_data.csv`.
