# Практика 2. MLP для класифікації якості білого вина

## Датасет

Використовується датасет **Wine Quality** з UCI Machine Learning Repository.

Джерело: https://archive.ics.uci.edu/dataset/186/wine%2Bquality

У цій практиці використовується частина про **біле вино**: 4898 зразків і 11 числових фізико-хімічних ознак.

Цільова змінна: `quality`.

Щоб зробити задачу зручною для класифікації, перетворіть `quality` на 3 класи:

- `0` - нижча якість: `quality <= 5`;
- `1` - середня якість: `quality == 6`;
- `2` - вища якість: `quality >= 7`.

## Бізнес-контекст

Уявіть, що виноробня хоче швидко оцінювати якість партії білого вина за лабораторними вимірюваннями. Модель не замінює дегустацію, але може допомогти попередньо віднести зразок до нижчої, середньої або вищої категорії якості.

## Що потрібно зробити

### 1. Підготовка даних

У notebook:

1. Завантажте CSV у `pandas`.
2. Перевірте розмір датасету, назви колонок і розподіл `quality`.
3. Створіть нову цільову змінну `quality_class` з трьома класами.
4. Розділіть дані на `X` та `y`.
5. Зробіть `train/test split` у пропорції `80/20` з `random_state=42`.
6. Масштабуйте ознаки через `StandardScaler`.

### 2. PyTorch-тензори

Перетворіть дані на PyTorch-тензори:

- `X_train`, `X_test` -> `torch.float32`;
- `y_train`, `y_test` -> `torch.long`.

### 3. MLP через nn.Sequential

Побудуйте модель:

```text
Linear(11 -> 24)
ReLU
Linear(24 -> 12)
ReLU
Linear(12 -> 3)
```

Для задачі використовуйте:

- функцію втрат `nn.CrossEntropyLoss`;
- змінну з функцією втрат назвіть `loss_fn`;
- оптимізатор `torch.optim.Adam`;
- `learning_rate=0.001`;
- 120-180 епох.

### 4. Train loop

Реалізуйте базовий train loop:

1. `model.train()`;
2. forward pass для всього `X_train_tensor`;
3. loss для всього `y_train_tensor`;
4. `optimizer.zero_grad()`;
5. `loss.backward()`;
6. `optimizer.step()`;
7. збереження train loss за епоху.

### 5. Оцінка якості

На test set порахуйте:

- `accuracy`;
- `f1_score` з `average="weighted"`;
- `confusion_matrix`.

Додайте короткий висновок:

- чи модель краще розрізняє середню якість або крайні класи;
- що видно з confusion matrix;
- чи достатньо стабільно зменшується train loss.
