# Практика 4. MLP для перевірки справжності банкнот

## Датасет

Використовується датасет **Banknote Authentication** з UCI Machine Learning Repository.

Джерело: https://archive.ics.uci.edu/ml/datasets/banknote%2Bauthentication

У датасеті є 1372 зразки банкнот. Для кожного зразка доступні 4 числові ознаки, отримані після обробки зображення банкноти:

- `variance`;
- `skewness`;
- `curtosis`;
- `entropy`.

Цільова змінна: `class`.

## Бізнес-контекст

Уявіть, що банк хоче автоматично перевіряти банкноти за ознаками, отриманими з image processing. Модель має прогнозувати, чи є банкнота справжньою або підробленою.

## Що потрібно зробити

### 1. Підготовка даних

У notebook:

1. Завантажте CSV у `pandas`.
2. Перевірте розмір датасету, назви колонок і баланс класів.
3. Розділіть дані на `X` та `y`.
4. Зробіть `train/test split` у пропорції `80/20` з `random_state=42`.
5. Масштабуйте ознаки через `StandardScaler`.

### 2. PyTorch-тензори

Перетворіть дані на PyTorch-тензори:

- `X_train`, `X_test` -> `torch.float32`;
- `y_train`, `y_test` -> `torch.long`.

### 3. MLP через nn.Sequential

Побудуйте модель:

```text
Linear(4 -> 12)
ReLU
Linear(12 -> 6)
ReLU
Linear(6 -> 2)
```

Для задачі використовуйте:

- функцію втрат `nn.CrossEntropyLoss`;
- змінну з функцією втрат назвіть `loss_fn`;
- оптимізатор `torch.optim.Adam`;
- `learning_rate=0.003`;
- 200 епох.

### 4. Train loop

Реалізуйте базовий train loop:

1. `model.train()`;
2. forward pass для всього `X_train_tensor`;
3. loss для всього `y_train_tensor`;
4. `optimizer.zero_grad()`;
5. `loss.backward()`;
6. `optimizer.step()`;
7. збереження train loss за епоху.

### 5. Оцінка якості

На test set порахуйте:

- `accuracy`;
- `f1_score`;
- `confusion_matrix`.

Додайте короткий висновок:

- чи навчилася модель відрізняти класи;
- які помилки видно з confusion matrix;
- чи достатньо стабільно зменшується train loss.
