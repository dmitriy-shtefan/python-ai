# Завдання 2: AI Team Lead

## Контекст

Ви працюєте з навчальним Django-проєктом https://github.com/AnnaGorova/02-online-quiz-system
Це застосунок для проходження квізів: користувач бачить список квізів, вводить ім'я, відповідає на питання, переглядає результат та історію проходжень.

Ваша команда — не розробники, а **AI Team Lead**. Ваш результат залежить від того, наскільки добре ви організуєте роботу coding agent: від постановки задачі до перевірки й виправлення зауважень.

**Не пишіть код руками.** Усі зміни коду, тестів і документації має виконати агент за вашими промптами. Ви аналізуєте, уточнюєте вимоги, приймаєте рішення та проводите review.

## Підготовка

1. Зробіть fork або клон репозиторію.
2. Скопіюйте підготовлений файл `AGENTS.md` у корінь клонованого проєкту.
3. Відкрийте `AGENTS.md` і переконайтеся, що coding agent прочитає його перед початком роботи.
4. Виконайте User Story з переліку нижче.

`AGENTS.md` — постійна інструкція для агента. Не потрібно дублювати його технічні правила в кожному prompt.

## User Stories

### Story A. Зрозумілий каталог квізів

**Як користувач, я хочу** швидко зрозуміти, який квіз обрати на головній сторінці, **щоб** не відкривати кожен із них по черзі.

Оновіть UI каталогу: картка квізу має виразно показувати назву, короткий опис, кількість питань і зрозумілу дію для старту. Передбачте приємний стан, коли квізів ще немає, та коректне відображення на вузькому екрані.

### Story B. Спокійний старт квізу

**Як учасник, я хочу** перед початком побачити чітке пояснення квізу й коректно ввести своє ім'я, **щоб** розуміти, що відбудеться далі.

Оновіть екран деталей квізу: зробіть інформацію про квіз і форму введення імені зрозумілими, додайте помітну валідацію помилки та доступний стан фокусу. Не змінюйте механіку збереження імені в сесії.

### Story C. Зручне проходження тесту

**Як учасник, я хочу** чітко бачити прогрес і свої вибрані відповіді під час проходження квізу, **щоб** випадково не пропустити питання та впевнено завершити тест.

Оновіть UI сторінки проходження: покажіть прогрес у форматі «відповіли X з Y», візуально відрізняйте вибраний варіант, зробіть кнопку завершення зрозумілою. Значення прогресу має оновлюватися в браузері без зміни логіки підрахунку результату на сервері.

## Ваше завдання як AI Team Lead

Пройдіть усі п'ять етапів у зазначеному порядку.

### 1. Планування

Складіть перший prompt для агента. Попросіть його:

1. прочитати `AGENTS.md` і релевантні файли проєкту;
2. проаналізувати вашу User Story;
3. запропонувати покроковий план, перелік файлів для змін, підхід до тестів і ручної UI-перевірки;
4. **не змінювати код**, доки команда не затвердить план.

Перевірте план: чи покриває він усі критерії Story, чи не розширює scope, чи враховує мобільний екран і доступність. За потреби поставте уточнювальні питання. Лише після цього дайте агенту дозвіл на реалізацію.

### 2. Реалізація

Окремим prompt доручіть агенту реалізувати затверджений план. Попросіть після роботи показати короткий звіт: змінені файли, реалізовані сценарії та команди, які він запускав.

Не виправляйте код самостійно, навіть якщо помітили помилку.

### 3. Тести

Окремим prompt попросіть агента проаналізувати реалізацію й додати або оновити тести для поведінки, яку можна перевірити автоматично. Попросіть запустити `python manage.py test` та чесно повідомити результат.

### 4. Code review

Окремим prompt попросіть агента виконати code review власного diff відносно User Story та `AGENTS.md`. Нехай він:

1. знайде проблеми у функціональності, UI/UX, доступності, адаптивності, тестах і зайвих змінах;
2. наведе зауваження з пріоритетом і посиланням на файл/рядок;
3. **не вносить виправлень** на цьому етапі.

Потім проведіть власний review: перегляньте diff, відкрийте застосунок у браузері й перевірте основний сценарій Story. Порівняйте свої висновки з review агента.

### 5. Виправлення

Надішліть агенту погоджений список зауважень. Попросіть виправити лише їх, оновити за потреби тести та повторно запустити перевірку. Перегляньте фінальний diff і результат тестів.

## На що звернути увагу під час review

- Чи збережена наявна логіка проходження й підрахунку квізу?
- Чи зрозумілий користувачеві наступний крок без пояснення викладача?
- Чи працює сценарій із порожніми даними, довгими назвами або відсутніми результатами?
- Чи можна пройти інтерфейс клавіатурою, чи мають поля видимий focus state?
- Чи залишається UI читабельним на мобільному екрані?
- Чи тести перевіряють поведінку, а не лише існування сторінки?
- Чи не додав агент непотрібні залежності або зміни поза Story?

## Що здати

1. Файл `ai-team-lead.md` у корені проєкту, у якому є:
   - отримана User Story;
   - ваш prompt для планування та план агента;
   - prompt на реалізацію;
   - prompt на тести та результат `python manage.py test`;
   - prompt на code review, зауваження агента й ваші додаткові зауваження;
   - prompt на виправлення та короткий підсумок фінальної перевірки;
   - 2–3 речення: яке ваше рішення як Team Lead найбільше покращило результат.
2. Один або два скріншоти готового UI: основний сценарій вашої Story та, за можливості, мобільний вигляд або важливий empty/error state.

## Як оцінюється робота

- якість постановки задачі та планування;
- послідовність роботи: реалізація → тести → review → виправлення;
- якість фінального UI та відповідність User Story;
- змістовний `ai-team-lead.md` і докази перевірки.
