# Домашнє завдання 19. CrewAI-застосунок для бізнес-сценарію

**Дата заняття:** 14.07.2026 (вт)  
**Модуль:** Модуль 8. Використання наявних рішень

## Мета

На основі CrewAI- та Streamlit-застосунку, розібраного на занятті, реалізувати аналогічний multi-agent workflow для іншої предметної області.

## Завдання

Використайте практику з заняття як основу та реалізуйте **один** із двох запропонованих сценаріїв:

1. **B2B Sales Leads** — за технічним завданням `crewai_task_sales_leads.md`.
2. **Customer Support** — за технічним завданням `crewai_task_support.md`.

### Вимоги до застосунку

1. Адаптуйте агентів, завдання, тексти інтерфейсу та фінальний результат під обрану предметну область.
2. Використайте `Agent`, `Task`, `Crew` і `Process.sequential`.
3. Передавайте результати попередніх завдань наступним через `context`.
4. Додайте введення даних користувачем, вибір OpenRouter model і temperature, а також кнопку запуску workflow.
5. Отримуйте `OPENROUTER_API_KEY` із змінних середовища або Streamlit secrets. Не зберігайте API key у коді чи repository.
6. Покажіть фінальний результат у Streamlit як Markdown і додайте можливість завантажити його у `.md`-файлі.
7. Перевірте, що застосунок запускається та формує результат, який відповідає acceptance criteria обраної специфікації.

## Що здати

- `app.py` з реалізованим CrewAI- та Streamlit-застосунком.
- Короткий `README.md` з описом обраного сценарію, інструкцією запуску та результатом перевірки.
