# Вправа: детермінований код чи інтелект

Для кожного прикладу визначте, який підхід доречніший: **deterministic** (детермінований код) чи **intellectual (probabilistic)** (інтелектуальний / ймовірнісний компонент, наприклад LLM-агент).

|  № | Приклад | Категорія     |
|---:|---|---------------|
|  1 | Скласти короткий підсумок дзвінка з клієнтом на основі розшифровки розмови. | Probabilistic |
|  2 | Провести щоденне закриття каси: зіставити платежі з POS-системи, банківського еквайрингу й повернення товарів, порахувати очікуваний залишок, зафіксувати розбіжності та сформувати звіт за затвердженим шаблоном. | Deterministic |
|  3 | Визначити тональність відгуку клієнта: позитивна, нейтральна чи негативна. | Probabilistic |
|  4 | Провести автоматичну перевірку платежу перед виплатою: звірити реквізити отримувача з форматом країни, перевірити ліміти за сумою та кількістю операцій, порівняти дані з переліком санкцій і направити платіж на ручну перевірку лише за формально заданими умовами. | Deterministic |
|  5 | Обробити нове замовлення: перевірити наявність кожного товару, застосувати промокод за заданими правилами, розрахувати доставку за регіоном і сумою замовлення, а тоді створити рахунок або повідомити про конкретну помилку. | Deterministic |
|  6 | Відібрати з резюме кандидатів тих, чий досвід найбільше відповідає опису вакансії. | Probabilistic |
|  7 | Обробити заявку на відпустку: звірити кількість запитаних днів із доступним балансом, перевірити перетин із уже погодженими відпустками команди, визначити потрібного керівника для погодження та змінити статус заявки після його відповіді. | Deterministic |
|  8 | Синхронізувати каталог товарів між двома системами: визначити нові, змінені та видалені позиції за ID і часом оновлення, передати потрібні зміни через API, повторити тимчасово невдалі запити за заданою політикою та записати журнал операцій. |               |
|  9 | Відповісти на запитання користувача про умови повернення товару, спираючись на правила магазину. |               |
| 10 | Запропонувати наступний найкращий крок для менеджера після аналізу листування з клієнтом. |               |
| 11 | Розпізнати, чи містить завантажене фото пошкоджений товар, і описати тип пошкодження. |               |
| 12 | Щогодини завантажувати CSV-файл із продажами, перевіряти структуру й типи даних, відхиляти дублікати та некоректні рядки, записувати валідні дані до бази й надсилати звіт про помилки відповідальній людині. |               |
