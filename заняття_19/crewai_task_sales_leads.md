# Технічне завдання

## 1. Мета

Створити Streamlit-застосунок, який демонструє роботу CrewAI на бізнес-прикладі: команда AI-агентів готує план пошуку та залучення потенційних клієнтів для продукту, який продають компаніям (B2B).

Послідовність роботи:

```text
Профіль клієнта -> Канали пошуку -> Перше звернення -> Оцінка клієнта -> План продажів
```

## 2. Сценарій користувача

1. Користувач вводить опис B2B-продукту або послуги.
2. Обирає модель OpenRouter і значення `temperature`.
3. Натискає кнопку запуску.
4. CrewAI послідовно створює портрет ідеального клієнта, добирає канали пошуку, пише тексти для першого звернення, визначає критерії оцінювання клієнтів і готує підсумковий план продажів.
5. Користувач бачить результат у Markdown і може завантажити файл.

## 3. Агенти

| Змінна агента | Роль | Мета | Опис ролі |
|---|---|---|---|
| `icp_researcher` | `Дослідник клієнтів` | Визначити портрет ідеального клієнта для B2B-продукту. | Ви досліджуєте потенційних клієнтів і перетворюєте опис продукту на зрозумілий портрет: тип компанії, відповідальні працівники, проблеми, потреби та причини придбати продукт. |
| `lead_planner` | `Фахівець із пошуку клієнтів` | Підібрати ефективні канали пошуку потенційних клієнтів. | Ви плануєте пошук клієнтів. Ви враховуєте, наскільки клієнт підходить продукту, доступність каналів, вартість контакту та конкретні дії для першого звернення. |
| `outreach_writer` | `Автор повідомлень` | Створити короткі повідомлення для першого звернення до потенційного клієнта. | Ви пишете персоналізовані повідомлення для LinkedIn та електронної пошти — коротко, конкретно й без нав'язливого продажу. |
| `sales_reviewer` | `Фахівець з оцінювання клієнтів` | Перевірити план пошуку клієнтів на якість, практичність і можливі ризики. | Ви перевіряєте план продажів. Ви помічаєте нечіткий портрет клієнта, невдалі канали пошуку, розмиті критерії оцінювання та ситуації, у яких команда може витратити час на клієнтів, яким продукт не підходить. |

## 4. Завдання агентів

| Змінна завдання | Агент | Опис | Очікуваний результат |
|---|---|---|---|
| `icp_task` | `icp_researcher` | На основі опису продукту визнач, які компанії можуть бути найкращими клієнтами. Опиши їхні галузі та розмір, працівників, які ухвалюють рішення про купівлю, основні проблеми, потреби й причини придбати продукт. | Розділ Markdown українською з пунктами: Портрет ідеального клієнта, Типові компанії, Хто ухвалює рішення, Проблеми й потреби, Причини купівлі. |
| `channels_task` | `lead_planner` | На основі портрета ідеального клієнта запропонуй канали пошуку потенційних клієнтів. Для кожного каналу поясни, кого саме там шукати, як використовувати канал і за яким показником оцінювати ефективність. | Markdown-таблиця: Канал \| Кого шукати \| Як використовувати \| Показник успіху. |
| `outreach_task` | `outreach_writer` | Підготуй короткі тексти для першого звернення до потенційного клієнта. Тексти мають бути конкретними, ненав'язливими й пов'язаними з проблемами клієнта, описаними в його портреті. | Розділ Markdown із готовими текстами: повідомлення в LinkedIn, тема листа, текст першого листа, повторне повідомлення. |
| `qualification_task` | `sales_reviewer` | Визнач, як команда продажів може відрізняти перспективних клієнтів від тих, кому продукт, імовірно, не підходить. Опиши критерії оцінювання, ознаки зацікавленості та ризики помилкової оцінки. | Markdown-таблиця: Критерій \| Ознака перспективного клієнта \| Ризик, якщо ігнорувати. |
| `final_task` | `icp_researcher` | Збери підсумковий план пошуку та залучення клієнтів на основі попередніх результатів. План має бути коротким, структурованим і зручним для команди продажів. | Markdown із розділами: Портрет ідеального клієнта, Канали пошуку, Тексти повідомлень, Оцінювання клієнтів, Показники успіху. |

## 5. Критерії готовності

| Критерій | Очікувана поведінка |
|---|---|
| API-ключ | Застосунок працює з `OPENROUTER_API_KEY`. |
| CrewAI | Усі завдання виконуються послідовно через `Process.sequential`. |
| Контекст | Наступні завдання використовують результати попередніх. |
| Результат | Користувач отримує практичний план продажів у форматі Markdown. |
| Тематика | Усі агенти й завдання працюють над пошуком і залученням клієнтів для B2B-продукту. |
